1. Introduction

This Vision Document defines the high-level requirements, scope, and overall direction of the MyFriend – Machine Learning Psychology project. It aims to consolidate the product vision, target users, key features, and constraints, serving as a foundational reference for development and future evolution of the system.

1.1 Purpose of the Vision Document

The purpose of this document is to collect, analyze, and define high-level user needs and product features. It provides a shared understanding of the system among stakeholders and serves as a guide for development decisions.

1.2  Product Overview

MyFriend is a desktop-based software application designed to provide initial psychological guidance through a conversational interface. The system will be distributed as an executable file (.exe) and will utilize structured data and lightweight artificial intelligence techniques. The initial version will include: • Text-based interaction with users • Predefined knowledge base from reliable sources • Basic interpretation of user input • Recommendation to seek professional help

1.3  References

• Academic psychology materials (books, articles, university content) • Software engineering concepts learned during undergraduate studies • Documentation practices based on standard software engineering literature

2. User Description

The system is intended for individuals seeking accessible psychological guidance without immediate access to professional services.

2.1 User/Market Demographics

• Individuals with limited financial resources • Users without access to quality psychological services • Young adults and students interested in self-development

2.2 User Profiles

• Users experiencing emotional or psychological discomfort • Individuals seeking guidance or clarification about personal issues • Users interested in self-reflection and mental well-being

2.3 User Environment

• Personal computers with basic hardware configurations • Windows operating system • Internet access (recommended for full functionality)

2.4 Key User Needs

• Accessible psychological guidance • Privacy and anonymity • Fast and simple interaction • Reliable and non-harmful responses • Encouragement to seek professional support

2.5 Alternatives and Competition

•Traditional psychological therapy • Public healthcare psychological services • Online content (blogs, videos, forums) • Existing chatbot-based assistants

3. Product Overview

3.1 Product Perspective

The system will operate as a standalone desktop application with optional internet connectivity. Architecture overview: User → Interface → Processing Logic (Python) → Database → External APIs
